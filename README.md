## About

I build **production systems that hold money, state, and hardware** — not demos.

My work sits where most tutorials stop: multi-tier escrow ledgers that must never double-release, atomic slot locking that survives a thundering herd, offline-first desktop clients that reconcile a local SQLite write queue against a Postgres source of truth, and thermal printers that need bytes on a serial port at 2 AM when the shop is busy.

- **Currently** — architecting **Cubical**, a two-sided freelance marketplace for Pakistan's premier graduates, with in-DB escrow, WebRTC voice/video, and Claude-powered proposal drafting.
- **Founder** at **Smarties & Cookies Technologies**, shipping multi-tenant SaaS to real businesses with real revenue.
- **Specialty** — the unglamorous half: concurrency correctness, idempotency keys, session revocation, graceful degradation, and deploy pipelines that fail loudly instead of quietly.
- **Philosophy** — the database is the source of truth for concurrency. Unique constraints beat application locks. Money is an integer, never a float.

## How I Build

```
Correctness first    The DB enforces concurrency. Unique constraints over app locks.
                     Money is integer paisa. Prices always come from the server.

Degrade gracefully   Redis down? Chat disables, the app still boots and serves.
                     No SMTP? Email is inert, local dev is unblocked.

Verify at the edge   Zod on every write. Admin roles re-read from the DB per request,
                     never trusted from a long-lived token.

Own the pipeline     Self-hosted runners, PM2 clusters behind Caddy, health-gated
                     deploys that fail loudly. Nightly pg_dump, verified by test restore.

Ship organic         No mock data in production. No fake success toasts.
                     Every UI action hits a real endpoint or it does not ship.
```
